1. Introducción

Este tema trata dos bloques clave en HTML: Tablas — para presentar datos en filas y columnas; y Formularios — para recopilar y enviar datos del usuario a un servidor.

Objetivo: entender las etiquetas esenciales (<table>, <form>, controles) y atributos importantes (p. ej. colspan, rowspan, action, method).

2. Tablas

2.1 Estructura típica

Una tabla en HTML puede contener (opcionalmente) <caption>, <thead>, <tbody> y <tfoot>. Las filas son <tr>, las celdas de datos <td> y las celdas de cabecera <th>.

Ejemplo: estructura básica de tabla

<table>
  <caption>Mi tabla de ejemplo</caption>
  <thead>
    <tr><th>Nombre</th><th>Edad</th></tr>
  </thead>
  <tbody>
    <tr><td>Ana</td><td>28</td></tr>
    <tr><td>Luis</td><td>33</td></tr>
  </tbody>
  <tfoot>
    <tr><td colspan="2">Total: 2 registros</td></tr>
  </tfoot>
</table>

2.2 <col> y <colgroup>

Para agrupar columnas y aplicar estilos específicos a columnas concretas se usan <colgroup> y <col>. Útiles cuando quieres colorear o dar anchos distintos a columnas sin tocar cada celda.

2.3 Atributos colspan y rowspan

colspan une celdas horizontalmente; rowspan las une verticalmente. El valor numérico indica cuántas celdas se fusionan.

Ejemplo: colspan y rowspan

<table class="demo">
  <tr>
    <th colspan="3">Encabezado combinado (3 columnas)</th>
  </tr>
  <tr>
    <td rowspan="2">Celda vertical larga (ocupa 2 filas)</td>
    <td>A1</td>
    <td>A2</td>
  </tr>
  <tr>
    <td>B1</td>
    <td>B2</td>
  </tr>
</table>

3. Formularios

3.1 Elemento <form> y atributos action y method

<form action="URL_de_procesamiento" method="get|post">. action indica el script/URL que recibirá los datos; method define cómo se envían:

  • GET: los datos se añaden a la URL (visibles). Úsalo para datos no sensibles y búsquedas.
  • POST: los datos van en el cuerpo de la petición (no visibles en URL). Úsalo para credenciales o datos privados.

3.2 Atributos comunes de controles

Los controles suelen usar: type, name, value, required, maxlength, readonly, disabled, autofocus, tabindex, etc.

3.3 Controles (resumen y ejemplos)

Tipos habituales:

  • Botones: <input type="submit">, <input type="reset">, <input type="button">
  • Cajas de texto: <input type="text">, <input type="password">, y multi-línea <textarea>
  • Checkboxes (type="checkbox") y radios (type="radio")
  • Listas desplegables: <select><option>
  • Agrupación de controles: <fieldset> con <legend>

Formulario de ejemplo (GET y POST; varios controles)

<!--
  Formulario de ejemplo:
  - action apunta a "proceso.php" (simulado)
  - method puede ser "get" o "post"
-->

<form action="/proceso.php" method="post">
  <fieldset>
    <legend>Datos personales</legend>

    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre" required maxlength="50" autofocus /><br>

    <label for="pass">Contraseña:</label>
    <input type="password" id="pass" name="clave" required /><br>

    <label>Intereses:</label><br>
    <input type="checkbox" id="web" name="intereses[]" value="web" checked />
    <label for="web">Desarrollo web</label>
    <input type="checkbox" id="diseno" name="intereses[]" value="diseno" />
    <label for="diseno">Diseño</label><br>

    <label for="curso">Curso preferido:</label>
    <select id="curso" name="curso">
      <option value="html">HTML (20h)</option>
      <option value="css">CSS (15h)</option>
      <option value="js">Javascript (20h)</option>
    </select><br>

    <label for="coment">Comentario:</label><br>
    <textarea id="coment" name="coment" rows="4" cols="40"></textarea><br>

    <input type="submit" value="Enviar" />
    <input type="reset" value="Borrar" />
  </fieldset>
</form>

4. Casos prácticos (resumen)

4.1 Caso práctico 1 — "Actividades formativas"

Consiste en crear una tabla con: título "ACTIVIDADES FORMATIVAS", cabecera con nombre del curso, horario y horas, varias filas con cursos (HTML, CSS, Javascript, Jquery) y en el pie el total de horas.

Solución básica (tabla con total)

<table class="demo">
  <caption>ACTIVIDADES FORMATIVAS</caption>
  <thead>
    <tr><th>Curso</th><th>Horario</th><th>Horas</th></tr>
  </thead>
  <tbody>
    <tr><td>HTML</td><td>16:00 - 20:00</td><td>20</td></tr>
    <tr><td>CSS</td><td>9:00 - 12:00</td><td>15</td></tr>
    <tr><td>Javascript</td><td>16:00 - 20:00</td><td>20</td></tr>
    <tr><td>Jquery</td><td>16:00 - 20:00</td><td>20</td></tr>
  </tbody>
  <tfoot>
    <tr><td colspan="2">Total horas</td><td>75</td></tr>
  </tfoot>
</table>

4.2 Caso práctico 2 — "Grupo de controles"

Uso de <fieldset> y <legend> para agrupar controles visualmente (por ejemplo datos personales, preferencias, etc.). El PDF trae la solución completa en los anexos. :contentReference[oaicite:3]{index=3}

5. Resumen y buenas prácticas

  • Usa <thead>, <tbody> y <tfoot> cuando necesites agrupar y semántica clara.
  • Prefiere POST para datos sensibles; GET para búsquedas o filtros.
  • Usa name en todos los controles para que sus valores se envíen correctamente al servidor.
  • Aplica required, maxlength y validación (HTML5 y/o JavaScript) para mejorar la calidad de los datos.
  • Usa <fieldset> y <legend> para accesibilidad y organización visual de formularios.
